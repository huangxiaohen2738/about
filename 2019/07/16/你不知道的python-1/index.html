Python 3.5 前后字典发生的变化

python3.5之前(包括python2) 如果想要字典保持插入时候的顺序（key顺序）, 必须要使用OrderedDict, 即

但是，在python3.6以后，会发现，即使使用默认的字典，迭代的时候也会保持插入时候的顺序，这是怎么回事呢？
细心一点还会发现，在迭代的时候速度会更快(尤其是字典较大的时候更明显)，占用内存空间更小。

如果你经常关注PEP的更新，会看到从python3.6起，参考pypy的字典实现，CPython修改了字典底层的实现，内存使用减少了20%到25%。
这对广大python使用者来说无疑是一个好消息，那么它又是如何实现的呢?

我们先从python如何初始化一个字典来说起。

python3.6 之前!!!!!

当我们初始化一个空字典的时候，CPython的底层会初始化一个8 * 3 的二维数组:

大致是这样的

当我们加入一条数据的时候 example_dict[“name”] = “a”

为什么在index=1的地方插入数据呢？
因为 hash(“name”) % 8 = 1 (数组index，每台电脑得出的值是不一样的)

这样初始化一个空字典大约需要8byte * 3 * 8 = 192byte

而在python3.6之后!!!!!!

变成了:

当一个长度为2的字典，现在只需要8byte(indexs) + 8 * 3 * 2 = 56byte

而在3.6之前, 仍然需要 3* 8 * 8 = 192byte, 减少了70%

并且因为index可以记录插入的顺序，所以现在的字典可以保证插入字典的先后顺序(即实现了orderedDict的功能)
